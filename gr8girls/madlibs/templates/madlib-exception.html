Solving errors

Here are some common errors and how to solve them:

ImportError
This means that the web site was unable to find your Madlib program to read. Make sure that your program is in a file in the public_html directory called MyMadLibs.py.
AttributeError
This will show up when you attempt to view a madlib which doesn't exist. Go back home to see an updated list of madlibs.
SyntaxError
This means that the website found your Madlib program to read, but didn't understand a line in the program. The error will be on the line number mentioned in the error message. Common mistakes include:
  • Not balancing quotes around text. If you start an open quote, be sure to end it at the right spot!
  • Mis-spelling a word.
  • Missing a comma in between items in a list. For example, for creating new blanks to fill in, each line should end with a comma. If this is the case, the line number actually causing the error will likely be the one before the one in the error message.
IndentationError
This means that you used an "if" or other statement ending in a colon without indenting in the next line. For example 
if blanks['name'] == andrew:
print "Your name is andrew!"
is incorrect, while 
if blanks['name'] == andrew:
    print "Your name is andrew!"
is fine.
KeyError
This means that you used a blank in your Madlib that was never set. This can happen if you make a mistake in reading the "blanks" variable. For example, blanks['animal'] is different from blanks['animals'].
